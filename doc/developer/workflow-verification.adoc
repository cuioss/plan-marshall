= Workflow Verification
:nofooter:
:toc: left
:toclevels: 2

xref:../../README.md[Plan Marshall] » xref:index.adoc[Developer Guide]

Workflow output verification is provided by the opt-in `plan-retrospective` skill rather than a dedicated `/verify-workflow` command. The structural check scripts (`collect-artifacts.py` and `verify-structure.py`) live inside `plan-retrospective` as aspect scripts.

== What it verifies

Verifies that workflow outputs (solution outline, references, tasks) are correct and complete. Classical assertions cannot work because LLM outputs vary in wording while semantic correctness matters.

== Architecture

[source]
----
Phase 1: Structural Checks (Script)     Phase 2: Semantic Assessment (LLM)
┌─────────────────────────┐            ┌─────────────────────────┐
│ • File existence        │            │ • Scope correctness     │
│ • TOON/MD validation    │────────────│ • Completeness check    │
│ • Required sections     │            │ • Decision quality      │
│ • Cross-references      │            │ • Intent alignment      │
└─────────────────────────┘            └─────────────────────────┘
----

Key principle: uses `manage-*` tool interfaces, not direct filesystem access.

== Usage

Verification is invoked as an opt-in retrospective at the end of a plan:

[source]
----
Skill: plan-marshall:plan-retrospective
----

See the skill's `SKILL.md` for aspect selection, invocation patterns, and report structure.
