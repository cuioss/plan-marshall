= Workflow Verification System
:toc:
:toc-placement: preamble

Workflow output verification is now provided by the opt-in `plan-retrospective` skill rather than a dedicated `/verify-workflow` command. The structural check scripts (`collect-artifacts.py` and `verify-structure.py`) have been absorbed into `plan-retrospective` as aspect scripts.

== Overview

Verifies that workflow outputs (solution_outline.md, references.json, tasks) are correct and complete. Classical assertions cannot work because LLM outputs vary in wording while semantic correctness matters.

[cols="1,3"]
|===
| Skill | `marketplace/bundles/plan-marshall/skills/plan-retrospective/SKILL.md`
| Aspects | `artifact-consistency`, `request-result-alignment`, `log-analysis`, `script-failure-analysis`, `permission-prompt-analysis`
| Scripts | `marketplace/bundles/plan-marshall/skills/plan-retrospective/scripts/`
|===

== Architecture

----
Phase 1: Structural Checks (Script)     Phase 2: Semantic Assessment (LLM)
┌─────────────────────────┐            ┌─────────────────────────┐
│ • File existence        │            │ • Scope correctness     │
│ • TOON/MD validation    │────────────│ • Completeness check    │
│ • Required sections     │            │ • Decision quality      │
│ • Cross-references      │            │ • Intent alignment      │
└─────────────────────────┘            └─────────────────────────┘
----

*Key principle*: Uses manage-* tool interfaces, not direct filesystem access.

== Usage

Verification is invoked as an opt-in retrospective at the end of a plan:

[source]
----
Skill: plan-marshall:plan-retrospective
----

See the skill's SKILL.md for aspect selection, invocation patterns, and report structure.
