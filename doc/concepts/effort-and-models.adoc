= Effort and Models
:nofooter:
:toc: left
:toclevels: 2

How Plan Marshall selects which model and effort level runs each LLM-judgement workflow.

Project authors configure `models.roles.<phase>[.<subkey>]` (hierarchical kebab-case keys like `phase-3-outline.research`, `phase-6-finalize.verification-feedback`) in `.plan/marshal.json`. The Claude target emits six `execution-context[-{level}].md` agent variants for the sole role-eligible agent. Dispatch sites resolve the target via `manage-config effort resolve-target --phase <phase> [--role <subkey>]` and call the matching variant by name. End users get the right model per role at dispatch time without any user-side build step or Claude Code restart.

== Three Layers

[cols="1,3", options="header"]
|===
|Layer |Purpose

|Levels
|Six ordinals (`low`, `medium`, `high`, `xhigh`, `xxhigh`, `max`) plus the `inherit` sentinel. Each ordinal maps to a single `(model, effort)` primitive pair. Defined once in `plan-marshall/standards/effort-levels.md`.

|Roles
|Phase-scoped registry of LLM-judgement workflows that may be pinned to a level. Six groups (`phase-1-init` … `phase-6-finalize`), each polymorphic with optional `default`, `research`, `verification-feedback`, `post-run-review` sub-keys per phase. Defined in `plan-marshall/standards/effort-roles.md`.

|Variants
|The build target emits one canonical + five suffixed agent files per role-eligible agent. The resolver picks which variant a dispatch calls. Emission rules in `extension-api/standards/ext-point-dynamic-level-executor.md`.
|===
