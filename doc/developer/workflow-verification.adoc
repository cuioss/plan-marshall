Workflow output verification is provided by the opt-in `plan-retrospective` skill rather than a dedicated `/verify-workflow` command. The structural and analytic aspect scripts (eight today: `collect-plan-artifacts.py`, `collect-fragments.py`, `check-artifact-consistency.py`, `check-manifest-consistency.py`, `analyze-logs.py`, `direct-gh-glab-usage.py`, `summarize-invariants.py`, `compile-report.py`) live inside `plan-retrospective/scripts/`.

== What it verifies

Verifies that workflow outputs (solution outline, references, tasks) are correct and complete. Classical assertions cannot work because LLM outputs vary in wording while semantic correctness matters.

[cols="1,3"]
|===
| Skill | link:../../marketplace/bundles/plan-marshall/skills/plan-retrospective/SKILL.md[`plan-retrospective/SKILL.md`]
| Aspects | The canonical aspect list (and which are conditional) lives in `plan-retrospective/SKILL.md` § Aspects. The synthesised report dispatches the LLM-judgement aspects in one envelope.
| Scripts | link:../../marketplace/bundles/plan-marshall/skills/plan-retrospective/scripts/[`plan-retrospective/scripts/`]
|===

== Architecture

[source]
----
Phase 1: Structural Checks (Script)     Phase 2: Semantic Assessment (LLM)
┌─────────────────────────┐            ┌─────────────────────────┐
│ • File existence        │            │ • Scope correctness     │
│ • TOON/MD validation    │────────────│ • Completeness check    │
│ • Required sections     │            │ • Decision quality      │
│ • Cross-references      │            │ • Intent alignment      │
└─────────────────────────┘            └─────────────────────────┘
----

Key principle: uses `manage-*` tool interfaces, not direct filesystem access.

== Usage

Verification is invoked as an opt-in retrospective at the end of a plan:

[source]
----
Skill: plan-marshall:plan-retrospective
----

See the skill's `SKILL.md` for aspect selection, invocation patterns, and report structure.
